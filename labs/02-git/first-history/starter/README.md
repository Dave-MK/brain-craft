# Lab: First History

Lesson: `git-init-and-commits`

This folder has a `digital_twin/` package with no version history yet.

## Task

1. Run `git init` in this folder.
2. Stage and commit `digital_twin/` with a message that explains **why**
   this package exists, not just what's in it.
3. Make one real change to `digital_twin/classification.py` (e.g. adjust
   `warning_kw`'s default) and commit it with a message explaining why
   you changed it.
4. Run `python test_cases.py` to check your work.

There is no `setup_repo.py` for this lab — initializing the repository
from scratch is the entire point of the exercise.
